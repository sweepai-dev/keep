## Basics

The New Relic provider allows you to manage your New Relic resources.

## Inputs

The New Relic provider requires the following inputs: `apiKey`, `accountId`. The `apiKey` is your New Relic API key and the `accountId` is your New Relic account ID.

## ProviderConfig

The New Relic provider has the following configuration options: `region`, `version`. The `region` is the region of your New Relic account and the `version` is the version of the New Relic API.

## Outputs

The New Relic provider provides the following outputs: `alerts`, `dashboards`. The `alerts` output is a list of your New Relic alerts and the `dashboards` output is a list of your New Relic dashboards.

## BaseProvider

The New Relic provider extends the BaseProvider. The BaseProvider provides basic functionality that all providers must have.

## Authentication Parameters

The New Relic provider requires the following authentication parameters: `apiKey`, `accountId`. The `apiKey` is your New Relic API key and the `accountId` is your New Relic account ID.

## Connecting with the Provider

To connect with the New Relic provider, you need to set your `apiKey` and `accountId` in the provider configuration. You can then use the provider to manage your New Relic resources.
